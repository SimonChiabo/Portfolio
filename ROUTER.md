# ROUTER

Strict routing index. No knowledge lives here — only pointers.

## Protocol (mandatory, every work cycle)

1. **Before acting:** Read `STATE.md` to understand the current context and focus.
2. Load **only** the authority your task needs (table below). Do not load the whole project.
3. **After acting:** Update the affected authority and `STATE.md`. The cycle is two-way; skipping this step rots the system.

## Routes

| Topic | File | Read when… |
|---|---|---|
| Current state, focus, open flags | `STATE.md` | Starting any work cycle; checking what is broken or pending |
| Why decisions were made; what was discarded | `DECISIONS.md` | Tempted to re-litigate, revise, or extend a decision |
| Project execution plan | `PORTFOLIO_PLAYBOOK.md` | Reviewing the overall roadmap and phases of the portfolio |
| Project content | `content/proyectos/*.mdx` | Adding or editing portfolio projects |
